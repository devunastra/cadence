Tech Stack

Backend
Next.js API Routes
Server-side only. All GHL, Retell & Supabase calls live here — never from the browser.
Database · Auth · Realtime
Supabase (Free tier)
PostgreSQL · Row-Level Security (RLS) · Supabase Auth · Realtime subscriptions
Hosting
Vercel
Built by the Next.js team — best integration, fastest cold starts for serverless API routes. Free tier sufficient for 2–3 studios.
Roles & Permissions
Super Admin
Developers / Agency
Bypasses all RLS. Creates studios, creates accounts, assigns studios to owners. Views all data across all studios. Internal use only — not visible to clients.
Studio Owner
Dance Studio Owners
Inherits all Staff permissions, plus: sees only their own studios, creates new studio locations, invites/removes staff. Settings: Business Profile + My Profile + My Staff.
Studio Staff
Front Desk / Coaches
Can view and edit leads, view call analytics + calendar (read-only), use the unibox (send & receive). Settings: My Profile only. Business Profile and My Staff are hidden entirely.

Leads Phase 1
Notion-style table view. All Notion fields replicated (name, status, level, action, phone, email, last contacted, first lesson, comments, source, reason, checkboxes, etc.). Inline editing. Seeded via one-time Notion import. Live updates via GHL new-contact webhook → Supabase.
Call Analytics Phase 2
KPI cards: call count, pickup rate, success rate, transfer rate, voicemail rate, avg duration, sentiment donut, disconnected reason donut. Global date range filter: presets (Today → All time) + custom picker (dual calendar, HH:MM:SS time inputs, Between / >= / <= modes). Data from Retell post-call webhook → Supabase.
Conversations (Unibox) Phase 3
SMS + Email via GHL Conversations API. GHL webhook → Supabase → Supabase Realtime → browser instantly. Two-panel layout: conversation list + message thread. Send & receive.
Calendar Phase 2
Read-only. Pulls live from GHL Calendar API (5-min server-side cache). No local Supabase copy — GHL is source of truth. No create/edit/delete.
Settings Phase 1
Business Profile (Owner only) — studio name, location, logo, GHL sub-account ID, Retell agent ID.
My Profile (all roles) — name, email, password.
My Staff (Owner only) — invite by email, assign role, remove. Scoped to current studio.

Data Flow
GHL new contact created → webhook → POST /api/webhooks/ghl-contact → Supabase leads table
GHL new message (SMS/Email) → webhook → POST /api/webhooks/ghl-message → Supabase messages table → Realtime → browser instantly
Retell call ends → post-call webhook → POST /api/webhooks/retell-call → Supabase calls table
Calendar page loaded → GET /api/calendar → GHL Calendar API (5-min cache) → browser (no Supabase copy)
⚠ All API keys (GHL, Retell) are server-side only. The browser never calls GHL or Retell directly.
Security Principles
✓ Supabase RLS on every table — users can only read/write their own studio's data
✓ Sensitive internal data on separate tables from user-editable data
✓ Rate limiting: auth 10/15min IP · send message 100/hr user · GHL calendar 5min cache · general 100/min user
✓ Webhook endpoints validate shared secret token before processing any payload
✓ All GHL / Retell calls server-side only — no secrets in frontend env vars
✓ Budget caps on Supabase + Vercel. RLS audit before each phase goes live.
